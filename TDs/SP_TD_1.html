STREAM PROCESSING : TRAVAUX PRATIQUES

OBJECTIFS

  • Mettre en place un environnement de développement.
  • Appréhender les outils installés.
  • Comprendre le fonctionnement de l’application.
  • Diagnostiquer et résoudre d’éventuels problèmes.

ETAPE 1: Installation des outils

JDK 8

  • Insérez le lien suivant dans votre navigateur : http://www.oracle.com/technetwork/java/javase/downloads/jdk8-downloads-2133151.html

  • Dans la partie “Java SE Development Kit 8u161”, cochez la case “Accept License Agreement”.
  • Choisissez la distribution de votre OS puis téléchargez le JDK correspondant.

  • Suivez les étapes jusqu’à installation complète du dossier dans votre répértoire de travail.

  • Vérifiez le bon déroulement de l’installation en tapant la commande suivante sur votre terminal :

VIRTUAL BOX

Pour toute information détaillée sur VirtualBox, reportez-vous au site web dédié : https://www.virtualbox.org/

  • Insérez le lien suivant dans votre navigateur : https://www.virtualbox.org/

  • Après avoir cliqué sur le bouton “Download Virtualbox 5.2”, téléchargez le package qui convient à votre système d’exploitation.

  • Enregistrez le fichier puis lancez l’éxecutable.

  • Choisissez les options d’installation qui vous conviennent puis lancez l’application.

  • Ouvrez les préférences de votre virtualbox.

  • Assurez vous d’avoir au moins 10 GO disponible dans le dossier d’installation de la VM.

VAGRANT

  • Naviguez vers le lien suivant puis choisissez le système d’exploitation compatible avec votre machine :
    https://www.vagrantup.com/downloads.html

  • Téléchargez et enregistrez l’exécutable de l’application.

  • Lancez le configurateur de l’application.

  • Acceptez les termes d’utilisation.

  • Choisissez le répértoire de destination pour l’installation.

  • Lancez l’installation en cliquant sur le bouton “Install”.

  • Après l’installation de tous les packages, cliquez sur “Finish”.

  • Le sytème vous demandera de redémarrer votre machine, cliquez sur “Yes”.

INTELLIJ IDEA

Pour toute information détaillée sur Intellij IDEA, reportez-vous au site web dédié : https://www.jetbrains.com/idea/

  • Insérez le lien suivant dans votre navigateur : https://www.jetbrains.com/idea/download/#section=windows

  • Choisissez la version “Community edition” relative à votre OS puis téléchargez l’application.

  • Lancez l’installation de l’application.

  • Choisissez la version compatible avec votre architecture machine (32 bits ou 64 bits) et associez les extensions selon votre convenance.

  • Lancez l’IDE après finalisation de l’installation.

  • Choisissez le thème selon votre convenance puis cliquez sur “Next Default Plugins”

  • Les plugins sont organisés par Type, cliquez directement sur “Next Featured Plugins”.

  • Installez le plugin relatif à Scala.

  • Cliquez ensuite sur “Start using Intellij IDEA”.

  • Créez un nouveau projet, nous allons utiliser Maven comme assembleur de l’application. Indiquez ensuite le dossier d’installation du JDK 8 pour assurer le fonctionnement de Maven.

  • Après avoir choisi “Next”, remplissez les différents champs qui identifieront le projet.

  • Après avoir cliqué sur “Next”, remplissez le nom champs “nom du projet” puis choisissez “Finish”.

  • Après le chargement du projet, choisissez l’option “Enable auto-import”.

  • Dans la vue projet, supprimez les dossiers source.

  • Créez un nouveau module Maven via clique droit sur le projet.

  • Après avoir choisi “Next”, donnez un nom au module puis cliquez sur “Next” à nouveau et enfin “Finish”.

  • Il faut maintenant indiquer à l’IDE qu’il s’agit d’un module Scala. Après un clique droit sur le module récemment créé, choissiez l’option “Add Framework Support”.

  • Cochez la case “Scala” puis cliquez sur le bouton “Create”.

  • Cliquez sur le bouton “Download” puis choisissez la version 2.11.8 de Scala et cliquez sur “OK”.

  • Après installation de la librarie, choisissez “Ok”

  • Créez un sous-répértoire “scala” dans le dossier “main” en cliquant droit sur ce dernier.

  • Marquez le sous-répértoire créé en tant que source via un clique droit.

  • Testez le bon déroulement de l’ensemble de la configuration en lançant le feuille de travail scala. Par un clique droit sur le sous-répértoire “scala”, choisissez “Scala Worksheet”.

  • Donnez un nom à votre feuille de travail.

  • Ecrivez du code scala pour en voir le résultat directement si votre syntaxe est correcte.

WINDOWS SEULEMENT

Cette section concerne uniquement les machines opérant sous Windows. Pour les machines opérant sous Linux ou Mac, veuillez passer à l’étape suivante.

  1. CHOCOLATEY

Pour toute information détaillée sur Chocolatey, reportez-vous au site web dédié : https://chocolatey.org

  • Lancez votre fenêtre de commande en mode “administrateur”, puis rentrez la commande ci-dessous et tapez “Entrée”.
@"%SystemRoot%\System32\WindowsPowerShell\v1.0\powershell.exe" -NoProfile -InputFormat None -ExecutionPolicy Bypass -Command "iex ((New-Object System.Net.WebClient).DownloadString('https://chocolatey.org/install.ps1'))" && SET "PATH=%PATH%;%ALLUSERSPROFILE%\chocolatey\bin"

  • Patientez jusqu’à la fin de l’installation puis vérifiez le bon déroulement du processus en rentrant la commande ci-dessous.
choco /?

  1. CYGWIN

Pour toute information détaillée sur Cygwin, reportez-vous au site web dédié : http://www.cygwin.com/

  • Téléchargez Cygwin en utilisant chocolatey via la commande ci-dessous.
choco install cyg-get

  • Après avoir rentré “Y” pour accepter l’installation, Cygwin s’installe automatiquement sur votre machine.

  • Rentrez une dernière fois l’option “Y” pour finaliser l’installation de Cygwin.

  1. PACKAGES
  • Installez le package SSH en utilisant Cygwin via la commande ci-dessous.
cyg-get openssh

  • Installez le package Rsync en utilisant Cygwin via la commande ci-dessous.
cyg-get rsync

  • Installez le package GIT en utilisant Cygwin via la commande ci-dessous.
cyg-get git

  1. TERMINAL
  • Lancez votre terminal Cygwin sous votre menu “démarrer”.

  • Vou devriez obtenir un terminal identique à la capture ci-dessous.

VAGRANT

  • Créez un répértoire dans votre disque principal nommé “boxes”

  • Dans votre terminal Cygwin naviguez vers le répértoire récemment créé.

  • Clonez le dossier vagrant à partir de github via la commande ci-dessous. Il contient le fichier configurant la VM qui tournera sous Virtualbox.
git clone https://github.com/badrebelabbess/StreamProcessing.git
  • Naviguez vers le dossier “Vagrant” contenant le makefile.

  • Entrez la commande suivante dans votre terminal Cygwin pour lancer la configuration de la VM.
vagrant up
  • La finalisation de la configuration indiquera que tous les composants sont lancés (zookeeper, spark, cassandra, etc).

  • En ouvrant virtualbox, vérifiez bien qu’une nouvelle VM a été créée sous le nom “vagrant_default_xxxx”.

  • A titre indicatif, il est possible de gérer la VM directement via virtualbox. Néanmoins, il est préférable d’utiliser les commandes de vagrant pour éviter certaines erreurs système en cas de mauvaise manipulation de la VM. Par exemple, pour éteindre la VM, préférez la commande ci-dessous au fait d’utiliser le bouton “Power Off” sur virtualbox.
vagrant halt
  • Pour vous connecter à la VM, utilisez la commande ssh suivante sous votre terminal Cygwin.
vagrant ssh
  • Le résultat est l’apparition d’un nouveau shell qui tourne sous un système Ubuntu.

  • Les services qui tournent dans votre nouvelle VM sont des conteneurs Docker. Vous pouvez le vérifier en entrant la commande “docker ps”.

  • Le dossier téléchargé contenant le makefile et les “fixes” a été monté directement sur la VM sous le dossier “vagrant”. Vous pouvez le vérifier en entrant la commande ci-dessous.

  • Créez un fichier texte “test.txt” sur votre machine hôté sité dans le dossier “StreamProcessing”.

  • Vous pouvez vérifier que votre fichier .txt a bien été monté directement sur la VM sans manipulatiion supplémentaire. Notez que la communication entre les deux dossiers est bi-directionnelle (hôte vers VM + VM vers hôte).

  • Mappez l’adresse locale de votre machine à celle de la nouvelle VM créée en rajoutant la ligne ci-dessous à la fin de votre fichier “hosts”. Pour cela, ouvrez le fichier “C:\Windows\System32\drivers\etc\hosts” dans un éditeur de texte en mode administrateur.
127.0.0.1 lambda-pluralsight
  • Sauvegardez votre fichier puis fermez le.

ETAPE 2: Familiarisation avec Zeppelin et Spark

Pour toute information détaillée sur Apache Zeppelin, reportez-vous au site web dédié : https://zeppelin.apache.org/

  • Lancez votre navigateur puis tapez l’url “localhost:8988”

  • Créez votre premier notebook “Spark Intro” en cliquant sur le boouton “Create new note” puis cliquez sur le bouton “Create Note”.

  • Pour information, le bouton de configuration en haut à droite liste les frameworks et langages reconnus par zeppelin par ordre de parsing.

  • Ecrivez le code suivant dans la console de zeppelin.
case class Item(itemName: String, itemNumber: Int)
val items = (0 to 100).map( i => Item(s"name-$i", i) )

val itemsRDD = sc.parallelize(items)
val itemsDF = itemsRDD.toDF
  • Lancez votre code via le bouton “RUN” situé tout à droite de l’écran ou via le raccourci SHIFT + ENTREE. Vous devriez obtenir le résultat ci-dessous après l’éxécution des instructions.
